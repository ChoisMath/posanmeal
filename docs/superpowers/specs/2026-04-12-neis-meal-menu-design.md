# NEIS 급식 식단 표시 기능 설계

**작성일**: 2026-04-12
**상태**: 승인됨

## 개요

NEIS 급식 API를 활용하여 포산고등학교의 일일 급식 식단(조식/중식/석식)을 학생·교사에게 표시하는 기능. 식단 탭을 모든 사용자 페이지의 첫 번째 탭으로 배치하여 로그인 시 식단부터 확인할 수 있도록 한다.

## 요구사항

### 기능 요구사항
- NEIS 급식 API로 포산고등학교 일일 식단 조회
- 석식 → 중식 → 조식 순서로 세 끼 모두 한 화면에 표시
- 기본 날짜는 오늘(KST 기준), 좌우 버튼 및 터치 스와이프로 날짜 이동
- 알레르기 정보(19종) + 칼로리 + 영양 정보 표시
- 학생/교사 페이지 모두 "식단" 탭을 첫 번째로 배치
- 교사의 개인석식/근무 QR을 하나의 "QR" 탭으로 통합

### 비기능 요구사항
- 서버 인메모리 캐싱 (날짜별, TTL 1시간) — API rate limit 절약
- 데이터 없는 날(주말/공휴일) 안내 메시지 표시
- NEIS API 오류 시 사용자 친화적 에러 메시지

## NEIS API 연동

### 엔드포인트
- URL: `https://open.neis.go.kr/hub/mealServiceDietInfo`
- 인증: `NEIS_API_KEY` 환경변수 (Railway에 설정됨)

### 요청 파라미터
| 파라미터 | 값 | 설명 |
|----------|-----|------|
| `KEY` | `process.env.NEIS_API_KEY` | API 인증키 |
| `Type` | `"json"` | 응답 형식 |
| `pIndex` | `1` | 페이지 번호 |
| `pSize` | `10` | 페이지 크기 |
| `ATPT_OFCDC_SC_CODE` | `"D10"` | 대구광역시교육청 |
| `SD_SCHUL_CODE` | `"7240189"` | 포산고등학교 |
| `MLSV_YMD` | `"YYYYMMDD"` | 조회 날짜 |

### 응답 파싱
- 경로: `data.mealServiceDietInfo[1].row`
- 식사 코드: `MMEAL_SC_CODE` — "1"(조식), "2"(중식), "3"(석식)
- 메뉴: `DDISH_NM` — `<br/>` 구분, 알레르기 코드 포함 (예: `쌀밥 (5.6.13)`)
- 칼로리: `CAL_INFO`
- 영양: `NTR_INFO` — `<br/>` 구분
- 에러 코드: `RESULT.CODE` === `"INFO-200"` → 데이터 없음

### 알레르기 코드 매핑 (19종)
| 코드 | 알레르기 | 코드 | 알레르기 |
|------|---------|------|---------|
| 1 | 난류 | 11 | 복숭아 |
| 2 | 우유 | 12 | 토마토 |
| 3 | 메밀 | 13 | 아황산류 |
| 4 | 땅콩 | 14 | 호두 |
| 5 | 대두 | 15 | 닭고기 |
| 6 | 밀 | 16 | 쇠고기 |
| 7 | 고등어 | 17 | 오징어 |
| 8 | 게 | 18 | 조개류 |
| 9 | 새우 | 19 | 잣 |
| 10 | 돼지고기 | | |

### 알레르기 파싱 정규식
```
/(.+?)\s*\(?([\d.]+)\)?$/
```
메뉴명에서 음식 이름과 알레르기 코드 번호를 분리.

## API 설계

### `GET /api/meals?date=YYYYMMDD`
- **인증**: 불필요 (공개 — 급식은 공공 데이터)
- **파라미터**: `date` (선택, 기본값: 오늘 KST `YYYYMMDD`)
- **캐싱**: 인메모리 `Map<string, { data, fetchedAt }>`, TTL 1시간

#### 성공 응답
```json
{
  "success": true,
  "date": "20260412",
  "meals": [
    {
      "type": "3",
      "typeName": "석식",
      "dishes": [
        { "name": "쌀밥", "allergies": ["5"] },
        { "name": "된장찌개", "allergies": ["5", "6", "13"] }
      ],
      "calories": "831.2 Kcal",
      "nutrition": "탄수화물(g):120.5\n단백질(g):35.2\n..."
    }
  ]
}
```

#### 데이터 없음 응답
```json
{
  "success": true,
  "date": "20260413",
  "meals": [],
  "message": "급식 정보가 없습니다"
}
```

#### 에러 응답
```json
{
  "success": false,
  "error": "급식 정보를 불러올 수 없습니다"
}
```

## NEIS 서비스 (`src/lib/neis-meal.ts`)

### 구조
```typescript
// 상수
const OFFICE_CODE = "D10";
const SCHOOL_CODE = "7240189";
const CACHE_TTL_MS = 60 * 60 * 1000; // 1시간

// 캐시
const cache = new Map<string, { data: MealResponse; fetchedAt: number }>();

// 알레르기 코드 매핑
const ALLERGY_MAP: Record<string, string> = { "1": "난류", "2": "우유", ... };

// 메인 함수
export async function fetchMeals(date: string): Promise<MealResponse>
```

### 캐시 로직
1. `cache.get(date)` 확인
2. 존재하고 `Date.now() - fetchedAt < CACHE_TTL_MS`이면 캐시 반환
3. 아니면 NEIS API 호출 → 파싱 → 캐시 저장 → 반환

### 메뉴 파싱 로직
1. `DDISH_NM`을 `<br/>`로 split
2. 각 항목에서 정규식으로 음식명 + 알레르기 코드 분리
3. 알레르기 코드를 `.`으로 split → 개별 코드 배열

## 컴포넌트 설계

### `MealMenu` (`src/components/MealMenu.tsx`)

클라이언트 컴포넌트. 학생/교사 페이지에서 공유.

#### 날짜 헤더
- 형식: `< 2026년 4월 12일 (일) >`
- 좌우 `<` `>` 버튼으로 날짜 이동
- 터치 스와이프: `touchstart/touchmove/touchend` — 50px 이상 이동 시 날짜 전환
- CSS `transition`으로 슬라이드 애니메이션

#### 식사 카드 (3개, 순서: 석식 → 중식 → 조식)
- **헤더**: 아이콘 + 식사명 + 칼로리
  - 석식: indigo 그라데이션, 달 아이콘
  - 중식: green 그라데이션, 접시 아이콘
  - 조식: amber 그라데이션, 태양 아이콘
- **메뉴 목록**: 음식명 + 알레르기 배지 (인라인, `text-xs`, 둥근 태그)
- **영양 정보**: 접이식 토글 (기본 닫힘)

#### 상태 처리
- 로딩: 스켈레톤 카드 3개
- 데이터 없음: "급식 정보가 없습니다" 안내
- 에러: "급식 정보를 불러올 수 없습니다. 잠시 후 다시 시도해주세요"

## 탭 구조 변경

### 학생 페이지 (`/student`)
**기존**: QR | 개인정보 | 확인 (3탭, 기본: QR)
**변경**: 식단 | QR | 개인정보 | 확인 (4탭, 기본: 식단)

### 교사 페이지 (`/teacher`)
**기존**: 개인석식 | 근무 | 확인 | (학생관리) | 개인정보 (4~5탭, 기본: 개인석식)
**변경**: 식단 | QR | 확인 | (학생관리) | 개인정보 (4~5탭, 기본: 식단)

#### QR 탭 통합 (교사)
- 탭 내부 상단에 세그먼트 컨트롤: `개인석식 / 근무`
- 선택에 따라 `QRGenerator type="PERSONAL"` 또는 `type="WORK"` 렌더링
- 기본값: 개인석식

### QR 토큰 발급 타이밍
- QR 탭 **진입 시** 토큰 발급 시작 (QRGenerator 마운트)
- QR 탭 **이탈 시** 타이머/갱신 중단 (QRGenerator 언마운트)
- 재진입 시 새 토큰 발급
